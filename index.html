JavaScript

Es un lenguaje de programación orientado a objetos, dialecto del estándar ECMAScript Fue desarrollado originalmente por Brendan Eich de Netscape con el nombre Mocha, posteriormente se le nombro Live Script, y finalmente JavaScript. El cambio de nombre coincido con el momento en el que Netscape agregó compatibilidad con la tecnología Java en su navegador en diciembre de 1995. El cambio de nombre dio la impresión de que el lenguaje es una prolongación de Java. Sin embargo, Java y JavaScript tienen semánticas diferentes. Se utiliza principalmente en su forma del lado del cliente (client-side), implementando como parte de un navegador web predeterminado mejoras en la interfaz de usuario y páginas web dinámicas, aunque existe una forma de JavaScript del lado del servidor. Desde el 2012, todos los navegadores modernos soportan completamente ECMAScript 5.1, una versión de JavaScript. Los navegadores más antiguos soportan por lo menos ECMAScript 3.

DOM

El Modelo de objetos de documento (DOM) y el W3C es una plataforma y una interfaz de lenguaje neutral que permite que los programas y los scripts accedan y actualicen dinámicamente el contenido, la estructura y el estilo de un documento El DOM está definido y administrado por W3C, por lo que los distintos navegadores simplemente aplican las especificaciones del World Wide Web Consortium, para dar soporte en sus aplicaciones. Cuando se carga una página web, el navegador crea un Modelo de objeto de documento de la página. Con el DOM, los programadores pueden construir documentos, navegar por su estructura, y añadir, modificar, o eliminar elementos y contenido. Se puede acceder a casi cualquier elemento HTML. El HTML DOM es un modelo de objetos estandar y una interfaz de programación para HTML que define:


  • -Los elementos HTML como objetos.
  • -Las propiedades de todos los elementos HTML.
  • -Los métodos para acceder a todos los elementos de HTML.
  • -Los eventos para todos los elementos HTML.

En otras palabras: el HTML DOM es un estándar sobre cómo obtener, cambiar, agregar o eliminar elementos HTML. DOM significa Document Object Model, en español seria Modelo de Objetos del Documento. El DOM es una interfaz que implementan los navegadores para poder interactuar con elementos HTML.

Referencias con JS

Hay dos formas de incluir JavaScript en HTML y hacer que funcionen juntos


  • Agregar JavaScript directamente a un archivo HTML

  • Para insertar JS de forma directa en HTML se utiliza la etiqueta <script> </script>, el código JS debe de quedar dentro de esta etiqueta.

    El código de JS se puede colocar dentro de : las etiquetas de <head>; y <body>.

    Dependiendo de dónde se agregue el código JS en el archivo HTML, la carga será diferente. Por lo general se recomienda agregarlo en el <head> para que permanezca separado del contenido HTML.

    Pero colocarlo dentro de <body> puede ayudar a mejorar la velocidad de carga, ya que el contenido del sitio web se carga más rápido, y solo después de eso se procesará el JS.


  • Agregar código JS a un archivo separadoL

  • Insertar JS en HTML directamente no siempre es la mejor opción. sobre todo si algunos script deben utilizarse en varias páginas, por lo tanto, es mejor mantener el código JS en archivos separados y hacer referencia a estos archivos de la misma forma en que se hace referencia a los documentos CSS.

    Algunos de los beneficios de agregar código JS en archivos separados son:

    • -Cuando los códigos HTML y JS están separados, se cumple el principio de diseño de separación y hace que todo sea mucho más sustentible y reutilizable.
    • -La legibilidad y el mantenimiento del código es mucho más fácil.
    • -Los archivos JS en caché mejoran el rendimiento general del sitio web al disminuir el tiempo que tendrán las páginas en cargarse.

Eventos

Un evento HTML puede ser algo que hace el navegador, o algo que hace un usuario.

JS puede “reaccionar”en estos eventos.


  • - onchange: Un elemento HTML ha sido cambiado
  • - onclick: El usuario hace click en un elemento HTML
  • - onmouseover: El usuario mueve el mouse sobre un elemento HTML
  • - onmouseout: El usuario se aleja el mouse de un elemento HTML
  • - onkeydown: El usuario presiona una tecla del teclado
  • - onload: El navegador ha terminado de cargar la página

Viewport

El viewport o ventana grafica es el area visible del usario de una pagina web. Varia segun el dispositivo y será más pequeña en un smartphone que en las computadoras.

Antes de las tabletas y los telefonos móviles, las páginas web se diseñaban solo para pantallas de computadora, y era común que las páginas web tuvieran un diseño estático y un tamaño fijo.

Para solucionar esto, los navegadores en esos dispositivos redujeron la página web completa para ajustarse a la pantalla.

Los usuarios están acostumbrados a desplazar sitios web verticalmente tanto en dispositivos de escritorio como móviles, pero no horizontalmente.

Por lo tanto, si el usario se ve obligado a desplazarse horizontalmente o alejarse, para ver toda la página web, se obtiene una experiencia de usuario pobre.


Etiqueta viewport

HTML5 introdujo un método para permitir que los diseñadores web tomen el control de la ventana gráfica, a travez de la etiqueta <meta>

Un elemento de vista <meta> le da al navegador instruciones sobre cómo controlar las dimensiones y la escala de la página.


<meta name “viewport” content = “width=divice-width, initial-scale=1.0>


La parte width = device-with establece el ancho de la página para seguir el sncho de la pantalla del dispositivo.

@medias

CSS2 introdujo tipos de medios. La regla @media, inreoducida en CSS2, hizo posible definir diferentes reglas de estilo para diferentes tipos de medios.

Se podian definir reglas de estilo para computadoras, impresoras, dispositivos de mano, televiciones, etc.

CSS3 introdijo media queries que extendieron la idea de los tipos de CSS2 en lugar de buscar un tipo de dispositivo, analizan la capacidad del dispositivo.

Las media queries se pueden utilizar para verificar muchas cosas, como: ancho y alto de la ventana grafica, ancho y alto del dispositivo, orientacion del dispositivo y la resolición.

El uso de media queries es una técnica popular para entregar una hoja de estilo personalizada para computadoras de escritorio, computadoreas portátiles, tabletas y télefonos móviles.
